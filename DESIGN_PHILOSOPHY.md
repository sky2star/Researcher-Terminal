# 🔬 Researcher Terminal —— 设计理念

### 一款基于认知经济学的科研任务管理工具

---

## 一、问题的起源：坐在电脑前的空白

每天早上醒来，坐到电脑桌前，我经常面对一个问题：

**「我今天要干什么？」**

脑子是空的。昨天做到哪里了？接下来该做什么？有哪些事情在等着我？

这个问题看似简单，却消耗着宝贵的认知资源。我需要花时间回忆、梳理、规划——而这些本该用来做真正的工作。

这个工具要解决的第一个问题就是：**让我坐下来就知道今天要干什么**。

---

## 二、核心洞察：注意力是稀缺资源

人的注意力极其有限。

认知科学告诉我们，工作记忆的容量大约只有 4±1 个组块。这意味着我们在任何时刻，**只能同时处理一两件事情**。

这就造成了一个根本性的冲突：

| 思考模式 | 内容 | 认知负荷 |
|----------|------|----------|
| **元认知** | "下一步做什么？整体进度如何？" | 高 |
| **执行认知** | "专注完成当前这一步" | 高 |

**两者很难同时进行。**

当你在执行一个任务时，如果还要同时记住"接下来还有哪些步骤"、"整体目标是什么"，注意力就会被分散。反之，当你在规划全局时，很难同时深入执行细节。

这就是为什么很多人会有这样的体验：做着做着就忘了下一步该做什么，或者想着全局就无法专注眼前。

---

## 三、设计哲学：规划与执行的分离

受游戏任务系统的启发，我设计了一个核心机制：**把规划和执行在时间上分离**。

### 📊 规划模式（Planning Mode）

**在你头脑清醒的时候**——比如早上刚开始工作，或者刚完成一个阶段性任务——进入规划模式。

这时你的任务是：

- 思考接下来要做什么
- 把复杂任务拆解成可执行的步骤（Steps）
- 为未来的自己设计好路线图

就像游戏里的任务系统：主线任务拆解成若干个子任务，每个子任务有明确的目标。

**例如**，训练一个模型可能需要这些步骤：

```
1. 准备数据集
2. 编写数据加载代码
3. 定义模型结构
4. 编写训练循环
5. 运行实验并记录结果
```

如果不记下来，我需要在执行每一步的同时，还要记住剩下的步骤。这非常消耗脑力。

但如果提前规划好，执行时我只需要看一眼列表，就知道下一步是什么。

### 执行时：专注当前步骤

当进入执行状态时，我不需要记住完整的地图。

我只需要：
1. 看一眼当前要做的步骤
2. 全身心投入完成它
3. 完成后打个勾，看下一个

**规划时思考全局，执行时专注局部。**

两种认知模式在时间上分离，互不干扰。

---

## 四、探索模式：借鉴开放世界游戏的地图系统

科研不总是有明确的路线图。很多时候我们面对的是：

**「我知道我要解决这个问题，但我不知道该怎么解决。」**

这时候没有步骤可以拆解，因为你还不知道路径是什么。

受开放世界游戏的启发，我设计了**探索模式（Exploring Mode）**。

### 🗺️ 开放世界的隐喻

在《塞尔达传说》或《艾尔登法环》这类游戏中，地图一开始是被迷雾笼罩的。你需要亲自去探索，才能逐渐揭开地图。

科研也是如此。**研究领域就是一张被迷雾覆盖的地图。**

- 你知道目的地大概在哪个方向
- 但具体的路径需要你自己去探索
- 每一次尝试都在揭开一点点迷雾
- 有时候走错路，但也因此发现了新的信息

### 📝 探索日志

探索模式的核心是**探索日志**：

- **记录每一次尝试**：做了什么？结果如何？
- **提取洞察**：这次尝试告诉了我什么？
- **标记突破**：哪些发现是关键性的？

这些日志的作用不只是记录，更是**帮助你梳理自己的研究地图**。

当你记录得足够多，回头看时，你会发现：
- 哪些路已经被证明走不通
- 哪些方向有希望
- 整体的研究版图逐渐清晰

**让头脑尽快恢复清晰，这是探索日志的核心价值。**

## 五、两种模式的切换

```
        ┌──────────────────────────────────────────┐
        │                                          │
        ▼                                          │
   ┌─────────┐    找到方法     ┌─────────┐         │
   │ 探索模式 │ ───────────→  │ 规划模式 │         │
   │         │                │         │         │
   │ 梳理地图 │                │ 拆解执行 │         │
   └─────────┘                └─────────┘         │
        ▲                          │              │
        │      遇到新的未知        │              │
        └──────────────────────────┘              │
                                                  │
                    完成任务 ─────────────────────┘
```

科研的真实过程就是在这两种模式之间不断切换：

1. 面对一个新问题，先进入**探索模式**，摸索方向
2. 当思路逐渐清晰，切换到**规划模式**，拆解执行
3. 执行过程中遇到新的未知，再切回**探索模式**
4. 循环往复，直到完成

## 六、与传统任务管理软件的区别

市面上有很多优秀的任务管理工具：Todoist、Notion、滴答清单、Things……它们都能帮助人们管理待办事项。

但当我尝试用它们来管理科研任务时，发现了一些根本性的不适配。

### 传统工具的隐含假设

| 假设 | 现实（科研场景） |
|------|------------------|
| 你知道要做什么 | 经常不知道 |
| 你知道怎么做 | 经常需要摸索 |
| 任务可以被拆解 | 很多时候无法拆解，因为路径未知 |
| 完成 = 打勾 | 探索的价值不在于"完成"，而在于"发现" |
| 失败 = 没完成 | 失败的尝试本身就是信息 |

传统工具的底层逻辑是**执行已知任务**。它们擅长帮你追踪"买菜、开会、交报告"这类目标明确、路径清晰的事项。

但科研的本质是**探索未知**。当你面对一个没有现成答案的问题时，传统的待办清单只会让你更焦虑——它提醒你"这件事还没完成"，却对"如何完成"无能为力。

### 核心区别对比

| 维度 | 传统任务管理 | Researcher Terminal |
|------|-------------|---------------------|
| **核心隐喻** | 清单（Checklist） | 地图（Map） |
| **默认状态** | 用户知道怎么做 | 用户可能不知道怎么做 |
| **失败的处理** | 任务未完成 = 失败 | 尝试未成功 = 信息积累 |
| **进度衡量** | 打勾数量 | 地图揭开的范围 |
| **适用场景** | 执行型任务 | 探索型 + 执行型任务 |
| **工作模式** | 单一模式 | 规划 / 探索 双模式 |

### 一个具体的例子

假设你的任务是「提升模型准确率」。

**传统工具的做法：**

```
☐ 提升模型准确率
```

然后呢？这个任务可能在你的清单里躺上一个月，每天看着它，却无从下手。

**Researcher Terminal 的做法：**

如果你知道怎么做 → **规划模式**：
```
☐ 提升模型准确率
  ├─ ☐ 分析当前错误样本
  ├─ ☐ 尝试数据增强
  ├─ ☐ 调整学习率
  └─ ☐ 对比实验结果
```

如果你不知道怎么做 → **探索模式**：
```
📝 探索日志：提升模型准确率

[01-10] 尝试了增大 batch size，效果不明显
        💡 洞察：可能不是优化问题

[01-11] 分析了错误样本，发现集中在某类数据
        ⭐ 突破：可能是数据分布的问题！

[01-12] 尝试对该类数据做过采样，准确率提升 2%
        💡 洞察：方向对了，继续深入

→ 找到思路，切换到规划模式，拆解执行步骤
```

### 不是替代，而是补充

传统的任务管理工具很好，它们在执行型场景中非常有效。

Researcher Terminal 要解决的是传统工具**覆盖不到的场景**：

- 当你不知道下一步做什么
- 当你不知道怎么做
- 当你需要在探索中逐渐找到方向
- 当你需要让大脑从"记忆步骤"中解放出来

**它不是更好的待办清单，而是一种不同的思维工具。**

## 七、设计原则

| 原则 | 说明 |
|------|------|
| **认知卸载** | 不要用大脑记忆步骤，把它们写下来 |
| **时间分离** | 规划的时间规划，执行的时间执行 |
| **快速恢复** | 每次坐下来，都能立刻知道该干什么 |
| **地图意识** | 通过记录探索过程，逐渐看清研究全貌 |
| **最小干扰** | 工具足够简单，不增加额外负担 |

这个工具首先是为我自己设计的。

它解决的是我每天面对的实际问题：
- 早上坐下来不知道干什么

- 面对复杂问题不知道从何下手
- 探索了很多但思路还是混乱

如果你也有类似的困扰，希望它对你也有帮助。
